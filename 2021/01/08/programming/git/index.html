git教程

发表于2021-01-08|更新于2025-01-08|编程

Git是目前世界上最先进的分布式版本控制系统。

操作流程

基本命令

git init 初始化，多出一个.git目录，这个目录是用来跟踪管理版本库的

git status 查看仓库当前状态，可以查看修改未提交的文件有哪些

git diff xxx.yyy xxx.yyy是文件，可以看到文件修改的内容

提交

git add xxx 添加文件到暂存区 git add --all 全部提交到暂存区

git commit -m "xxx" 提交到本地仓库，xxx为本次提交的说明

回退

git log 查看提交日志

git reset --hard HEAD^ 回退到上一个版本。上上一个版本就是HEAD^^，往上100个版本写100个^比较容易数不过来，所以写成HEAD~100。

--hard会回退到上个版本的已提交状态，而--soft会回退到上个版本的未提交状态，--mixed会回退到上个版本已添加但未提交的状态。

回退上一个版本后，git log无法看到最新的版本了，如果此时（命令行没关）要到在最新的版本，要找到commit id，git reset --hard 1094a，版本号写前几位，能确定唯一即可；如果命令行关掉了，用git reflog，这是用来记录每一次命令的，从输出可以得到commit id。

撤销

git checkout -- Readme.md 把Readme.md文件在工作区的修改全部撤销，这里有两种情况：

  • 一种是Readme.md自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
  • 一种是Readme.md已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

git reset HEAD <file>可以把暂存区的修改撤销掉,重新放回工作区

删除

在文件管理器中把没用的文件删了，工作区和版本库就不一致了，用git rm xxx从版本库里删掉

如果是误删，用git checkout -- xxx从版本库恢复到工作区

远程仓库

git remote add origin git@github.com:xxx.git 将本地仓库关联与远程仓库关联

git push -u origin main 将当前分支main推送到远程仓库；-u参数还会把本地的main分支和远程的main分支关联起来，在以后的推送或者拉取时就可以简化命令

git remote -v 查看远程库信息

git remote rm <name> 解除本地仓库和远程仓库的关联

git clone <url> 克隆远程仓库到本地

分支管理

git branch 查看分支

git branch xxx 创建分支

git switch xxx 切换分支

git switch -c xxx 创建并切换分支

git merge xxx 用于合并指定分支xxx到当前分支，如将dev分支合并到main分布git merge dev

git branch -d xxx 删除分支

标签管理

git tag 查看标签

git tag v1.0 打标签

git show <tagname> 查看标签信息

git tag -a v0.1 -m "version 0.1 released" <commit id> 创建带有说明的标签，用-a指定标签名，-m指定说明文字

如果忘记打标签了，可以git log --pretty=oneline --abbrev-commit查看commit id，git tag v0.9 <commit id>

git tag -d v0.1 删除标签

git push origin :refs/tags/v0.9 如果标签已经推送到远程,从远程删除的命令是push

git push origin <tagname> 推送某个标签到远程

git push origin --tags 一次性推送本地标签到远程
